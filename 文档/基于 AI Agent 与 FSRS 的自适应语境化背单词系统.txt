# 基于 AI Agent 与 FSRS 的自适应语境化背单词系统

## 毕业设计完整方案总结（最终修订版）

---

## 一、选题背景与研究动机

传统背单词软件（如墨墨背单词、扇贝等）普遍采用"单词↔中文释义"的记忆模式，通过用户主观点击"认识/不认识"来判断熟悉度。这种模式存在三个核心缺陷：

1.  **记忆模式机械**：检验的是词义映射，而非真实语境中的理解能力，导致"记住词义但不会用"的假性掌握；
2.  **评分维度单一**：仅由用户主观自我报告，易受社会赞许性或惰性影响，缺乏客观行为数据的支撑；
3.  **出题逻辑固定**：按固定题库轮排，无法针对个人薄弱点进行定向强化。

与此同时，间隔重复算法已从 SM-2 演进到 FSRS（Free Spaced Repetition Scheduler），后者通过难度（D）、稳定性（S）、可提取性（R）三个变量建模记忆状态，在学术研究和工业应用中均取得了更优的调度效果。然而，即便 FSRS 也仅考虑了历史复习结果，尚未将单次学习过程中的多维行为数据（如反应时长、跳过行为、提示求助等）融入遗忘模型。

大语言模型的涌现能力则为自适应学习带来了新的可能。利用 DeepSeek 等大模型构建的 AI Agent，不仅能够提供知识服务，更能扮演"备考教练"的角色——规划学习路径、诊断薄弱点、动态推荐学习模式。本研究旨在开发一套基于 Spring Boot + Vue3 的智能背单词系统，在 FSRS 算法基础上引入由多维行为数据驱动的动态遗忘因子（DF），并设计 AI Agent 作为顶层备考教练，串联**快速记忆模式**、**语境深度学习模式**与**统一复习模式**，实现真正意义上的自适应语境化学习。

---

## 二、核心创新点

1.  **多维行为数据替代主观评分**：系统自动采集答题正误、反应时长（前端本地计时，过滤异常值）、是否查看提示、是否跳过、键入时长及历史复现率等多维行为序列，映射为客观连续浮点评分（grade），无需用户自评。

2.  **FSRS + 动态遗忘因子（DF）**：在标准 FSRS 的稳定性更新环节，乘入由实时行为参数计算的动态遗忘因子 DF，使记忆间隔能够实时响应学习者的注意力波动、疲劳程度和历史表现。DF 参数参考 FSRS 原论文的参数范围与教育学常识进行预设，并通过消融实验验证有效性。为防止极端行为数据导致 DF 因子过度放大，对各独立 λ 因子设置值域约束。

3.  **多模态学习体系**：
    *   **快速记忆模式（Quick Memory）**：墨墨式极简流程，用于新词的初次快速接触，基于"认识/不认识"判断与闪电验证，配合错误强化机制，DF 同时纳入反应时与历史准确率。
    *   **语境深度学习模式（Contextual Deep Learning）**：面向新词深度学习或顽固词攻坚，采用五步递进流程（单词卡 → 词义选择 → 选词填空 → 短文填空 → 单词补全），在真实语境中进行记忆的精加工。步骤1的停留时长作为辅助行为信号纳入 DF。
    *   **统一复习模式（Unified Review）**：所有已学单词的巩固出口，采用三步检测（闪电判断 → 语境选义 → 词形辨析）并自动分流至条件追加拼写，实现复习分层。

4.  **Agent 备考教练（TiMo）**：Agent 不再局限于知识点问答，而是升格为横跨各学习模块的"备考教练"。它基于 FSRS 全局记忆状态和备考进度，主动完成**每日词单推荐、模式智能调度、顽固词诊断与干预、每周策略分析**。用户通过对话或采纳建议，即可实现从"用户自管"到"Agent 带学"的范式转变。

5.  **语境化题型体系 + Agent 动态出题**：在语境深度学习模式中，Agent 实时调用 DeepSeek 生成贴合备考目标的真题例句、短文填空和词义辨析，突破固定题库限制。

## 四、系统整体架构与功能模块全景

### 4.1 核心架构：教练-入口-模式

系统采用"**一个顶层教练 + 一个统一入口 + 三个执行模块 + 一套共享算法引擎**"的架构。

```
                    ┌─────────────────────────────────────┐
                    │       Agent 备考教练 (TiMo)          │
                    │  规划 · 推荐 · 诊断 · 调度 · 闭环    │
                    └─────────────────────────────────────┘
                           │          ▲          │
              ┌────────────┼──────────┼──────────┼────────────┐
              │            │          │          │            │
              ▼            ▼          │          ▼            ▼
       ┌──────────┐  ┌──────────┐     │    ┌──────────┐  ┌──────────┐
       │ 快速记忆  │  │语境深度  │     │    │ 统一复习  │  │ 备考规划  │
       │ (轻量接触) │  │ (深度学习)│     │    │ (巩固检测) │  │ (目标设定)│
       └──────────┘  └──────────┘     │    └──────────┘  └──────────┘
            │              │          │          │              │
            └──────────────┴──────────┴──────────┴──────────────┘
                                │
                     ┌──────────────────┐
                     │  FSRS + DF 引擎   │
                     │  (共享记忆状态)    │
                     └──────────────────┘
```

### 4.2 前端页面与组件设计

**整体布局**：顶部导航栏（今日规划、选词学习、复习待办、词库、统计、个人中心），响应式设计。右下角全局 Agent 悬浮按钮（TiMo小猫 + 淡蓝光环），点击展开对话中心。

**① 登录/注册页**：JWT 身份认证；注册后触发 TiMo 备考规划对话，通过对话收集用户的备考目标、历史考试成绩、可用时间等，生成个性化的动态备考计划。

**② 今日学习规划页（Agent 主导）**：顶部展示学习状态快照与 TiMo 每日问候。中部核心区展示 Agent 根据备考计划、FSRS 全局数据（到期词数、顽固词数）与用户意图生成的**今日学习推荐**，包含推荐新词数量、复习数量与建议学习模式（快速记忆或语境深度学习）。用户可一键采纳推荐词单，也可手动调整。

**③ 选词入口（统一准备层）**：提供灵活的单词筛选条件（词书、熟悉度、搜索），用户可手动勾选或全选/反选。右侧已选清单实时更新数量和单词列表。底部是学习方式切换与 TiMo 推荐卡片，用户选择以何种模式开始学习后，点击开始按钮分流至对应执行模块。

**④ 快速记忆模式页面**：沉浸式墨墨背单词风格。展示单词卡片（无中文释义），用户点击"认识"触发二选一闪电验证（反应时<3秒），点击"不认识"自动展开释义、真题例句与 TiMo 助记。错误单词在 Session 末尾强制重复验证。

**⑤ 语境深度学习模式页面**：分组学习，每组默认 10 个词。按顺序推进五个步骤：单词卡（熟悉音形义与真题考点）→ 词义选择（4选1）→ 选词填空（全组单词-释义匹配）→ 短文填空（Agent 生成融入全组单词的短文）→ 单词补全（渐进式字母提示）。一组完成后展示小结与薄弱词。

**⑥ 统一复习模式页面**：轻量快速，所有到期词统一排队。每个词依次进行三步检测：闪电判断（3秒二选一）→ 语境选义（5秒四选一）→ 词形辨析（5秒选正确拼写）。检测中表现不佳或已标记为顽固词者，自动追加拼写环节进行强化。

**⑦ 词库管理、统计仪表盘、个人中心、学习日历、错词本/生词本、词义辨析、AI周报、对话中心（TiMo）**等模块功能齐全，与上述核心学习模式深度联通。

## 五、核心算法：FSRS + 动态遗忘因子（DF）

### 5.1 FSRS 基础模型与多模式适配

FSRS 用三个参数描述卡片记忆状态：稳定性 (S)、难度 (D)、可提取性 (R)。

\[
R = \exp\left( \ln(0.9) \cdot \frac{\Delta t}{S} \right)
\]

当 R 跌至阈值（如 0.9），卡片进入复习队列。每次复习后，根据连续浮点评分 grade（1.0~4.0）更新 D 和 S。

本系统三种模式共享同一 FSRS 引擎，但采用不同的评分路径与 DF 因子组合：

| 模式 (study_mode) | 评分来源 | DF 因子构成 | 时间因子 λ_rt | 历史因子 λ_acc | 跳过/提示因子 λ_skip |
|-------------------|----------|-------------|---------------|----------------|----------------------|
| **快速记忆** (`quick_memory`) | 认识判断+验证映射 | `DF = λ_rt × λ_acc × 1.0` | 基于点击反应时 | 基于该词历史正确率 | 不启用（无跳过提示机制） |
| **语境深度学习** (`context_deep`) | 五步加权+主动输出惩罚 | `DF = 1.0 × λ_acc × λ_skip_custom` | 不启用（整组作答不可归因） | 基于各次 grade_final | 基于多步累计提示量 `hint_total` |
| **统一复习** (`unified_review`) | 三步检测综合评分 | `DF = λ_rt × λ_acc × 1.0` | 基于步骤1反应时 | 基于该词历史正确率 | 不启用（无跳过提示机制） |

### 5.2 各模式评分机制

#### 5.2.1 快速记忆模式评分

| 行为 | grade |
|------|-------|
| 认识 + 验证正确 | 4.0 |
| 认识 + 验证错误 | 2.0 |
| 不认识 | 1.0 |

#### 5.2.2 语境深度学习模式综合评分

该模式下每个单词经历步骤 2~5 的作答，产生四份原始成绩（s2~s5_raw，取值范围0-4）。步骤1（单词卡）为纯展示不作答，但其**停留时长**作为辅助行为信号：若停留时间不足 3 秒即点击"继续"，视为用户跳过阅读，该信号纳入 λ_skip_custom 的输入（在 hint_total 计算中附加 +1 惩罚项）。

加权合成 `composite_grade`，再经步骤5（主动输出）表现惩罚，得到最终传给 FSRS 的 `grade_final`。

\[
composite\_grade = 1.0 + 3.0 \times \left( 0.20\frac{s2}{4} + 0.25\frac{s3}{4} + 0.30\frac{s4}{4} + 0.25\frac{s5}{4} \right)
\]

\[
grade\_final = composite\_grade \times \left( 1 - 0.2 \cdot (1 - \frac{s5}{4}) \right)
\]

#### 5.2.3 统一复习模式三步检测评分

复习模式中，每个单词依次经历"闪电判断、语境选义、词形辨析"三步，每步独立评分（0~5分）。

\[
grade_{review} = 1.0 + 3.0 \times \left( 0.35\frac{step1}{5} + 0.30\frac{step2}{5} + 0.35\frac{step3}{5} \right)
\]

若 `grade_review < 2.5`，自动追加拼写环节，拼写结果作为额外字段记录，并触发更严厉的错误强化。

### 5.3 动态遗忘因子 DF

DF 在 S 更新后进行乘算修正：`S_final = S' × DF`，最终截断至 [0.5, 1.5]。

**各 λ 因子定义与值域约束**：

为防止个别极端行为数据（如异常快速或缓慢的作答）导致单个 λ 因子过度膨胀，进而频繁触发 DF 的整体截断上限，各独立 λ 因子在合成 DF 之前先行截断至 [0.7, 1.3]。此举保留了 DF 在正常行为范围内的精细调节能力，避免截断本身掩盖真实的行为差异。

*   **λ_rt (反应时因子)**：`λ_rt = 1 + θ₁·(1 - (t-μ)/(2σ))`，截断至 [0.7, 1.3]。参数 θ₁ 预设为 0.3。
*   **λ_acc (历史复现率因子)**：`λ_acc = 1 + θ₂·(p - 0.8)`，截断至 [0.7, 1.3]。参数 θ₂ 预设为 0.5。
*   **λ_skip_custom (语境深度的提示惩罚因子)**：`λ_skip_custom = 1 - θ₃·(hint_total/5)·(1+γ·review_count)`，截断至 [0.7, 1.3]。参数 θ₃ 预设为 0.2，γ 预设为 0.1。DF 合成前各因子值域均被约束，确保最终 DF 在合理范围内波动。

**参数来源说明**：
上述 θ₁~θ₃、γ 的预设值基于以下原则确定：(1) FSRS 原论文中 grade 对 S 更新影响的效应量范围（w 参数约 0.2~0.5）；(2) 认知心理学中反应时与记忆强度关系的文献经验（反应时每偏离均值 1σ，记忆强度波动约 15~30%）；(3) 教育学中提示使用对长期记忆的负面影响研究。在线实验中通过消融研究（对比开启/关闭 DF、对比不同参数组合）验证有效性，完整参数学习与离线仿真作为展望内容。

### 5.4 冷启动策略

| λ 因子 | 冷启动策略 |
|--------|-----------|
| **λ_rt** | 新用户 μ=8s、σ=3s（群体画像默认值），累计正确答题 ≥30 次后平滑过渡到个人统计值 |
| **λ_acc** | 新单词首次学习时 p 无历史数据，λ_acc 恒设为 1.0（无偏差） |
| **λ_skip_custom** | 新单词首次学习时 hint_total=0，代入公式后 λ_skip_custom = 1.0（无惩罚），无需额外处理 |

### 5.5 错误强化机制

所有模式均设有错误强化规则，用于在 FSRS 标准更新外对顽固问题进行补丁式修正：

| 模式 | 触发条件 | 操作 |
|------|---------|------|
| 快速记忆 | 本次 grade ≤ 2.0 | S × 0.9, D + 0.05 |
| 快速记忆 | 重复验证再次错误 | S × 0.8, D + 0.1, 标记顽固词 |
| 语境深度学习 | s5 ≤ 1 (拼写严重困难) | S × 0.8, D + 0.08 |
| 语境深度学习 | 连续两组均有步骤错误 | 标记顽固词 |
| 语境深度学习 | hint_total ≥ 4 (大量求助) | S × 0.85, 不增加 review_count |
| 统一复习 | 三步综合 grade < 2.0 | S × 0.85, D + 0.05 |
| 统一复习 | 追加拼写仍错误 | S × 0.8, D + 0.08, 标记顽固词 |
| 所有模式 | 顽固词在同一模式内连续 3 次 grade ≥ 3.5 | 解除顽固标记 |

### 5.6 对话练习与 FSRS 的数据隔离

用户在 Agent 对话中心进行自由对话练习时，若正确使用目标词造句，系统**不直接**生成 quiz_record 写入 FSRS 的主记录流。理由：对话场景与正式复习场景的认知负荷、上下文线索、时间压力均不同，混入同一条记录流会污染遗忘曲线的准确性。

改为：对话练习中成功使用目标词的行为**单独记入 `conversation_quiz_log` 表**。Agent 在推荐今日词单时，检查该表中的近期记录：若某单词 3 天内在对话中被正确使用过，则 Agent 为其打上临时标记 `conversation_mastered = true`，该标记使该词在本次选词推荐中被过滤（不进入推荐列表）。标记仅影响选词推荐的排序与展示，**不修改 FSRS 的 S 和 D 值**——这意味着该词仍需按 FSRS 原定计划进入复习，只是在用户主动选词时不会被重复推荐，避免了"已会用但系统还推"的厌烦感。临时标记有效期 3 天，过期自动清除，之后 Agent 恢复对该词的正常推荐逻辑。

## 六、AI Agent 备考教练（TiMo）机制

Agent 不参与单题教学，其职责是基于全局数据（FSRS、备考计划、历史行为）进行**顶层调度与闭环决策**。

### 6.1 串联场景

*   **场景1：顽固词深度学习推荐**：监测到某词在快速记忆模式连续错误，Agent 在 session 结束后主动弹出建议："这词需要深度学习，要不要切换到语境深度学习模式？"采纳后自动将词预选入深度学习选词入口。
*   **场景2：进度落后干预**：备考进度落后原计划 15%，Agent 在用户打开规划页时提醒，并建议增加语境深度学习模式攻坚高频词。
*   **场景3：模式成果闭环反馈**：语境深度学习攻克顽固词后，Agent 自动清除该词在快速记忆池中的顽固标记，并反馈至复习模式调整调度。

### 6.2 Agent 对话中心

右下角悬浮的 TiMo 对话框支持**多模式切换快捷指令**（"帮我推荐今天怎么学"、"我想深度学习这几个顽固词"），也支持**单词问答、词义辨析、学习诊断**等知识服务。

## 七、实验设计与评估方案

### 7.1 离线模拟实验（主要验证手段）

为适应毕业设计的时间约束，系统有效性验证以**离线模拟实验**为主。编写脚本模拟三种典型用户行为模式，在相同初始条件下对比不同算法的调度效果。

**虚拟学习数据生成规则**：

| 参数 | 生成规则 | 说明 |
|------|---------|------|
| 反应时 t | 服从正态分布 N(μ=8s, σ=3s)，截断至 [1s, 30s] | 基于文献中词汇判断任务的平均反应时设定 |
| 快速判断正确率 | 高频词（词频前 30%）p₀=0.85，中频词（30%~70%）p₀=0.70，低频词（后 30%）p₀=0.55 | 正确率按词频分层，模拟真实词汇认知规律 |
| 认识/不认识判断 | 基于该词当前 R 值：R≥0.9→90%点"认识"；R≥0.7→70%点"认识"；R<0.7→30%点"认识" | 用户认识判断与FSRS可提取性正相关 |
| 提示使用概率 | hint_total 服从泊松分布 λ=1.5，截断至 [0,5] | 低频词 λ 上浮至 2.0 |
| 单词卡停留时长 | N(12s, 5²)，截断至 [2s, 60s] | 模拟阅读行为 |

**对比指标**：
*   14 天后各单词的 R 值分布（记忆保持均匀度）
*   总复习次数（调度效率）
*   顽固词产生的数量与收敛速度

**消融分析**：对 DF 的三个 λ 因子做消融实验，量化每个因子对调度效果的独立贡献。

### 7.2 小规模用户评估（辅助验证手段）

招募 3~5 名同学进行为期 3~5 天的短期试用。试用结束后填写系统可用性量表（SUS），并对使用体验进行简短的开放式访谈。由于样本量小，本环节定位为**探索性用户反馈**，论文中以定性描述方式呈现结果（如"3名用户认为快速记忆模式节奏感好""用户普遍反馈顽固词提示有帮助"），不作统计显著性检验。该反馈用于佐证系统可用性，不构成主要的量化验证结论。

## 九、总结

本方案设计了一个由 AI Agent 贯串的智能备考背单词系统。系统以 FSRS 记忆模型为理论核心，创新性地引入多维行为数据驱动的动态遗忘因子，并设计**快速记忆、语境深度学习、统一复习**三种互补的学习模式。Agent 作为顶层备考教练，实现了"模式间智能路由"与"备考全周期数据闭环"，使系统从传统背词工具升级为主动规划、因材施教的 AI 学习伙伴。方案在学术深度、工程可行性与用户体验方面达成平衡，具备扎实的支撑与鲜明的创新特色。
